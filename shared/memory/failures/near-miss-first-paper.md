# Near-miss mining as first paper

- **Idea:** Start with monocular near-miss scenario mining.
- **Why it failed for now:** Validation burden is too high before establishing what reconstruction errors are acceptable. Rare events are exactly where monocular uncertainty is most dangerous.
- **Evidence / reasoning:** Good application story, bad first scientific wedge.
- **Revisit later?** Yes, after the error-budget story is established.
